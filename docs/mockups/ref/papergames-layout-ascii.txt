GAME PAGE PATTERN (e.g. Tic Tac Toe)
------------------------------------
Same header. Then:
- Game thumbnail + title + one-line tagline ("First to align three marks wins")
- App store badges
- Play modes: [ Play with a friend ] [ Play vs robot ] [ Create tournament ] [ Play online ]
- Daily leaderboard (countdown)
- Side/block: "Integrate this game" (API) link
- Body: game description, rules, similar games, strategies

STRUCTURE SUMMARY
-----------------
1. Header: logo + grouped nav (Gruppo, Games, Pricing, Resources) with dropdowns.
2. Value props: one headline + 3 badges (Free, Multiplayer, Child friendly).
3. Hero: main title + short copy + app store badges.
4. Feature grid: 4 blocks (ranking, play with friend, tournaments, shop) with headline + copy + Learn more.
5. Game grid: 6 cards (icon + name); link to each game.
6. Optional CTA strip: remove ads / premium plan.
7. Footer: social, resource links, legal, copyright.

TAKEAWAYS FOR URSA MINOR
------------------------
- Value-props strip gives quick trust (free, multiplayer, family-friendly) without clutter.
- Feature blocks explain how to play (share link, tournaments) and monetization (shop, ad-free).
- Game grid is primary; keep cards simple (icon + name) like Neal.fun but with optional one-liner.
- Game page: thumbnail + tagline + play-mode buttons + leaderboard/API hooks.
- Footer can stay minimal; social + legal + contact.
- Align with night-sky motif: calm, no purple gradients; badges and cards can feel like gentle stars.

# Shown after user clicks "Play" on a game page.
# Assumes most games use a square board; some need score and/or whose turn.
# References: play2048 (score, best, new game, 4x4 grid), NYT single (instruction, grid, controls)

STRUCTURE SUMMARY
-----------------
1. Header: Logo + nav only. Game chrome bar: game name + turn (or score) only; no actions in chrome.
2. Main: instruction, square board (NxN), info bubbles (scores/stats), action buttons (New game, Hint), controls hint.
3. Info bubbles sit close to the game (below board); action buttons sit below the bubbles.
4. Footer: horizon style.

GAME CHROME VARIANTS (bar above board; actions always below board)
------------------------------------------------------------------
| Game type     | Chrome bar              | Info bubbles (below board)   | Action buttons   |
|---------------|-------------------------|------------------------------|------------------|
| Puzzle/arcade | Game name               | Score, Best                  | New game, Hint   |
| Turn-based 2p | Game name, Whose turn   | X wins, Ties, O wins         | New game, Hint   |
| 2p + score    | Game name, Turn         | P1 score, P2 score (or pieces)| New game, Hint  |
| Solo puzzle   | Game name               | Score, Timer (optional)      | New game         |

REFERENCES APPLIED
------------------
- Tic-Tac-Toe (XO): Info bubbles (X YOU, Ties, O CPU) below board; turn + reset at top. We put bubbles below board, actions below bubbles.
- Reversi: Player/opponent piece counts above board; New game + Hint buttons below board.
- Tetris mockup: Stats in sidebar next to playfield; Pause/New game in header. We use bubbles + actions below for consistency.
- play2048: Score/Best; we show these in bubbles below board and move New game to action row.
- DESIGN_BIBLE: Calm, minimal. Board is focus; info and actions stay close and below.
